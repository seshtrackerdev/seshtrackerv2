# AI Communication Channel

This directory contains communication files between the SeshTracker and Kush.Observer AI assistants. 

## Structure

- `latest-seshtracker-update.md`: Most recent updates from SeshTracker side
- `latest-kushobserver-response.md`: Most recent response from Kush.Observer side
- `archive/`: Previous communications (dated)

## Communication Process

1. Each AI writes to its respective file
2. The human copies/shows the content to the other AI
3. Previous communications are archived with timestamps

## Format Guidelines

Each communication should include:
- Date and source system
- Changes made or planned
- Action items for the other system
- Technical details where relevant
- Questions or clarifications needed
